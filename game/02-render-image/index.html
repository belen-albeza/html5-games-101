The game state can access the Phaser.Game instance via this.game.

In the preload function we can load our image assets using the Phaser.Loader instance that is available in game.load. We supply a string key –we will use this later to reference the image– and the path to the asset.

Images are a game entity in Phaser: an instance of Phaser.Image. When an image is in our game world, it will automatically be rendered each frame. We can create an image and add it to this game world via game.add, which is a factory for multiple game entities. To do this, we call game.add.image and provide the coordinates and the image key –the one we used in preload.

game.add.image(0, 0, 'background'); // render image at position 0,0
